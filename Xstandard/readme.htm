XStandard Web Services For PHP Readme File

Introduction To XStandard Web Services

Web Services are applications that run on server computers and communicate with other computers using a dialect of XML called SOAP (Simple Object Access Protocol). Typically, business users do not interact directly with Web Services. Instead, they interact with user-friendly programs (such as XStandard) that in turn use Web Services. XStandard uses Web Services to check spelling, to upload files from the local computer to the server and to reference files located on remote servers.

XStandard Pro works with 5 Web Services - Spell Checker, Image Library, Attachment Library, Directory and Subdocument. Spell Checker is used for spell checking. Image Library is used to upload images from a local computer to the server and to build and browse image libraries. Attachment Library is used to upload files from a local computer to the server and to build and browse file libraries. Directory is used to communicate with 3rd party systems (like your CMS) and insert code snippets into the editor. Subdocument is used to replace custom elements with subdocuments.

Requirements

  • PHP 4.3.0+
  • PHP modules: zlib, pspell
  • Linux, FreeBSD or Windows NT+

Installation

  1. Unzip x-web-services-php.zip
  2. Move spellchecker.php, spellchecker.config, imagelibrary.php, attachmentlibrary.php, linklibrary.php, imagelibrary.config, attachmentlibrary.config, directory.php, directory.config, directory-example-product-1.txt, directory-example-product-2.txt, directory-example-product-3.txt, directory-example-product-4.txt, directory-xhtml-entities.xml, directory-xhtml-latin1.xml, directory-xhtml-special.xml, directory-xhtml-symbol.xml, directory_example_staff.csv, subdocument.php, subdocument-example-A.txt, subdocument-example-B.txt, subdocument-example-C.txt, subdocument-example-D.txt and subdocument-example-E.txt to a virtual root.

Testing

Open a Web browser and for each Web Service navigate to the URL where the Web Services is located. For example:
http://localhost/imagelibrary.php
http://localhost/attachmentlibrary.php
http://localhost/directory.php
http://localhost/spellchecker.php
http://localhost/subdocument.php
If you don't see Status: Ready in the browser, then the Web Service is not correctly installed. The most common cause of this is incorrectly set file permissions.

Launch test file test.htm. Specify the path to the license file and to Web Services. Click on the "Update" button. You should now be able to reference files from a library, drag an image into the editor, or spell check the editor's content.

Support

Additional dictionaries (German, French, Spanish, etc.) for the Spell Web Service are available from aspell.net. See online documentation for configuration details. Free technical support is available.
